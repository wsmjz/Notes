# redux

核心思想：视图派发action，reducer处理更新state，subscribe订阅数据变化

  • 与react无直接关系的 状态管理器
  • 一个仓库store,包含调度中心(reducer)与state(状态容器)
  • 提供disptch,getState,订阅器
  • 共享types, 在reducer和组件中都会使用
  • store的状态变了，不会刷新组件, 在组件中要订阅变化事件才行

# react-redux与vuex的差异

  • 和组件结合方式的差异。VUE通过VUEX全局插件的使用，结合将store传入根实例的过程，就可以使得store对象在运行时存在于任何vue组件中。而React-Redux则除了需要在较外层组件结构中使用以拿到store之外，还需要显式指定容器组件，即用connect包装一下该组件
